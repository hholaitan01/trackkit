1 SEND DRIVER LOCATION

// Get GPS from browser navigator.geolocation.watchPosition(pos => { // Send to TrackKit every 5 seconds fetch(`${API}/v1/deliveries/${id}/location`, { method: "POST", headers: { "Authorization": `Bearer ${apiKey}`, "Content-Type": "application/json" }, body: JSON.stringify({ lat: pos.coords.latitude, lng: pos.coords.longitude, speed: pos.coords.speed, heading: pos.coords.heading }) }) })
The API auto-calculates ETA, broadcasts location via WebSocket to all tracking widget subscribers, and triggers status transitions when close to the dropoff (e.g. DELIVERING → ARRIVING at <200m).

2 UPDATE DELIVERY STATUS

fetch(`${API}/v1/deliveries/${id}/status`, { method: "PATCH", headers: { "Authorization": `Bearer ${apiKey}`, "Content-Type": "application/json" }, body: JSON.stringify({ status: "DELIVERING" }) })
Valid transitions: PICKUP_EN_ROUTE → PICKED_UP → DELIVERING → DELIVERED. Status changes are broadcast via WebSocket and trigger webhook events.

3 LIST ACTIVE DELIVERIES

const res = await fetch(`${API}/v1/deliveries?status=DELIVERING`, { headers: { "Authorization": `Bearer ${apiKey}` } }) const { data } = await res.json() // data = [{ id, trackingCode, status, pickupAddress, ... }]
